## Политика сессии Humanoid / human-like активности

- Лимит времени на human-like действия (скролл/микро-движения/anti-sleep): 3 минуты на одну сессию/запрос и вкладку.
- Триггер старта: запуск пайплайна/запроса (например, start streaming/continuous activity).
- Истечение: по завершении 3 минут автоматически останавливаются циклические human-активности (ContinuousHumanActivity, drift/micro-scroll), но функционал расширения (копирование ответов, отправка следующих запросов) продолжает работать.
- Новый запрос в той же вкладке запускает новую сессию на 3 минуты; предыдущий таймер гасится.
- Механизм: в ContinuousHumanActivity добавлен sessionTimeoutMs (по умолчанию 3*60*1000) и stop(reason='session-timeout') при превышении.
